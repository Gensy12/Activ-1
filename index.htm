1. ¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, etc.) de un atributo o característica de una entidad. Por ejemplo, el número 25, la palabra "perro", la fecha 2024-07-20 son ejemplos de datos.

2. ¿Qué es información?

La información es el conjunto de datos organizados y procesados para que tengan significado. Por ejemplo, si tienes los datos de temperatura y humedad de un lugar durante un día, la información sería el análisis de estos datos para concluir si hizo calor o frío ese día.

3. ¿Diferencia entre dato e información?

La diferencia radica en el contexto y el significado. Los datos son los valores brutos, mientras que la información son los datos organizados y procesados que tienen relevancia y contexto para el usuario.

4. ¿Tipos de datos?

Los tipos de datos pueden ser numéricos (enteros, decimales), textuales (cadenas de caracteres), booleanos (verdadero/falso), fecha/hora, entre otros dependiendo del sistema de gestión de bases de datos.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo de datos para una base de datos. Este modelo incluye entidades, atributos y relaciones, y sirve como una representación visual y estructurada de cómo se organizarán y relacionarán los datos.

6. ¿Tipos de bases de Datos, según: - Variabilidad - Contenido - Modelo de base de datos

Variabilidad: Bases de datos estáticas (donde los datos no cambian frecuentemente) y dinámicas (donde los datos cambian con frecuencia). Contenido: Bases de datos de texto (como documentos), imágenes, multimedia, etc. Modelo de base de datos: Relacional, NoSQL (como documentales, de grafos, etc.).

7. ¿Qué es el modelo relacional?

El modelo relacional es un tipo de modelo de datos que organiza los datos en tablas relacionadas. Utiliza relaciones basadas en claves (generalmente llaves primarias y foráneas) para asociar los datos entre sí.

8. ¿Describa los elementos del Diagrama entidad Relacional?

Un diagrama entidad-relación incluye entidades (objetos o conceptos representados como rectángulos), atributos (características de las entidades representadas como óvalos) y relaciones (conexiones entre entidades representadas por líneas).

9. ¿Qué es una entidad?

Una entidad es un objeto o concepto distinguible del mundo real que se representa en una base de datos. Por ejemplo, una entidad puede ser una persona, un producto o una transacción

10. ¿Qué es un atributo y sus tipos?

Un atributo es una propiedad o característica de una entidad. Puede ser numérico, textual, booleano, fecha/hora, etc., dependiendo del tipo de dato que represente.

11. ¿Qué es una relación?

Una relación es una asociación significativa entre entidades. Por ejemplo, la relación "compra" entre una entidad "cliente" y una entidad "producto" indica que un cliente ha comprado un producto.

12. ¿Qué son las llaves primarias, foránea y sus características?

La llave primaria es un atributo o conjunto de atributos que identifica de manera única a cada registro en una tabla. La llave foránea es un atributo en una tabla que se relaciona con la llave primaria de otra tabla para establecer una relación entre ellas. Las características incluyen la unicidad (en el caso de la llave primaria) y la referencialidad (para la llave foránea).

13. ¿Qué son las reglas del Negocio?

Las reglas del negocio son restricciones o directrices que se aplican a los datos dentro de una organización. Estas reglas aseguran que los datos sean consistentes, precisos y relevantes para las operaciones de la empresa.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad en el contexto de las bases de datos indica la cantidad de instancias en una entidad que pueden estar relacionadas con una instancia en otra entidad. Puede ser uno a uno, uno a muchos o muchos a muchos. El mapeo se refiere a cómo se representan estas relaciones en un modelo de datos.
